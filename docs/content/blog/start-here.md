这篇文章是一个写作模板。以后你要发布新文章，只需要在 `docs/content/blog/` 目录下新建一个 Markdown 文件，例如：

```text
docs/content/blog/my-first-post.md
```

然后在文件开头写 front matter：

```yaml
---
title: "文章标题"
date: 2026-06-25
authors:
  - name: 你的名字
tags:
  - 技术
  - 复盘
---
```

正文直接使用 Markdown 编写即可。

## 推荐文章结构

一篇技术博客可以先按这个结构写：

1. 背景：为什么遇到这个问题
2. 过程：你尝试了什么，踩了什么坑
3. 结论：最后怎么解决
4. 复盘：下次怎么做得更好

不用一开始就追求完美。博客真正的价值，是把模糊的经验变成可以被再次使用的文字。
