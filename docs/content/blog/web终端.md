# 详细功能需求

## FR-01 Web SQL 终端与会话

**功能目标：**
在浏览器内提供受控的数据库连接、SQL 编辑、风险检测和执行能力。

**触发条件：**
运维人员登录数据库资源时，选择“Web 终端”登录方式。

**前置条件：**

- 用户已登录平台。
- 用户拥有目标数据库资源及数据库账号的访问权限。
- 数据库资源处于启用状态。
- 应用已发布为“Web 终端”类型。

**处理规则：**

- 页面由资源信息栏、数据库选择区、SQL 编辑区、执行工具栏、结果区和会话信息区组成。
- 数据库连接由服务端代理建立，数据库账号和密码不得传递至浏览器。
- 前端通过 HTTPS 和 WebSocket 与平台通信。
- 会话创建后显示资源名称、数据库名称、数据库账号别名、连接状态、会话时长和审计编号。
- 编辑器支持 SQL 高亮、行号、基础自动缩进、SQL 格式化、选中执行和快捷键 `Ctrl/Cmd+Enter`。
- 存在选中内容时，仅执行选中的 SQL。
- 无选中内容时，执行光标所在的完整 SQL 语句。
- “执行全部”作为独立按钮，执行前进行二次确认。
- 单次提交的 SQL 文本大小支持管理员配置，默认上限为 1 MB，可配置范围为 1-100 MB。
- 单次查询结果最多展示 5,000 行，超出部分不在页面展示，并提示“结果已截断”。
- 查询执行超时时间默认1分钟，管理员可在1-3600分钟范围内配置。
- 用户可以主动停止正在执行的查询，停止操作必须写入审计日志。
- 页面连续30分钟无操作时自动关闭会话（管理员可自定义），并提前1分钟提示用户。
- 用户退出平台、权限失效或资源被禁用时，服务端应关闭对应数据库连接。
- 同一会话内同一时间只允许执行一个SQL任务。
- 停止、超时或执行失败后，只终止当前任务，不关闭Web终端会话，阻断，终止任务，关闭回话。
- 数据库切换仅允许选择当前账号有权限访问的数据库。

**异常与提示：**

- 网络中断：页面显示“连接已断开”，允许用户手动重新连接，不自动重新执行SQL。
- 数据库连接断开：结果区展示数据库返回的错误信息，用户可手动重新连接。
- SQL文本超过限制：不提交执行，并提示当前上限。
- 查询结果超过5,000行：仅展示前5,000行，并提示“结果已截断”。
- 查询超时：终止当前查询，并提示“执行超过时间限制，已停止”。
- 用户权限失效：立即禁止新的SQL执行，并关闭当前会话。
- WebSocket心跳超时：服务端释放数据库连接，避免产生失效会话。

**日志要求：**

- 和现有状态保持一致

**验收标准：**

- 用户可以连接已授权资源并执行SELECT语句，结果以表格形式展示。
- 快捷键、执行按钮和选中执行的处理逻辑一致。
- 未选中SQL时，仅执行光标所在语句。
- SQL超时或被用户停止后，数据库侧查询同步终止。
- 当前任务停止后，用户可以继续执行下一条SQL。
- 网络断开后不会自动重复执行上一条SQL。
- 无权限用户不能创建或继续使用Web终端会话。

## FR-02 SQL脚本上传与执行

**功能目标：**
支持用户上传并执行`.sql`文件，同时确保脚本中的高危SQL不会被部分执行。

**触发条件：**
用户在Web终端中点击“上传脚本”。

**前置条件：**

- 用户已建立有效的Web终端会话。
- 用户具有目标数据库的SQL执行权限。
- 当前会话没有其他正在执行的任务。

**处理规则：**

- 仅支持`.sql`文本文件。
- 文件大小上限支持管理员配置，默认20 MB，最大不得超过100 MB。
- 文件编码仅支持UTF-8和UTF-8 BOM。
- 客户端和服务端必须分别校验文件类型、文件大小和文件内容。
- 上传完成后展示文件名、文件大小、文件哈希、语句数量和前1,000行内容，不立即执行。
- 系统按照MySQL语法完成SQL语句分割。
- 支持`DELIMITER`、存储过程、函数、触发器等复杂脚本语法（需要调研一下是否好做）。
- 系统必须先对脚本中的全部SQL完成权限校验、语法解析和风险预检。
- 任意一条SQL命中阻断规则时，整个脚本不执行，不得向数据库提交任何语句。
- 预检阻断时，页面最多展示100条命中记录，并显示剩余命中数量（最好支持自定义）。
- 命中记录包含语句序号、所在行号、风险类型、名单库名称。
- 全部SQL预检通过后，用户点击“确认执行”并进行二次确认。
- 脚本按照语句顺序逐条执行。
- 某条SQL执行失败后，默认停止执行后续SQL。
- 执行失败前已经成功执行的SQL不自动回滚。
- 脚本执行结果按照语句序号展示执行状态、影响行数、执行耗时和错误信息。
- 用户可以主动停止脚本，停止后不再执行后续语句。

**异常与提示：**

- 文件编码无法识别：提示“仅支持UTF-8编码，请转换后重新上传”。
- 文件超过配置上限：客户端和服务端均拒绝上传。
- 文件扩展名与实际内容不一致：拒绝上传。
- SQL语句分割失败：不执行脚本，并提示错误所在行号附近的内容。
- 风险预检服务异常：采用失败关闭策略，整个脚本不执行。
- 执行过程中数据库连接断开：停止后续SQL，并保留已完成语句的执行记录。
- 用户停止脚本：提示已执行语句数量和未执行语句数量。

**日志要求：**

保持现在逻辑

**验收标准：**

- 包含任意高危SQL的脚本不会向数据库提交任何语句。
- 预检通过的脚本按顺序执行。
- 某条SQL失败后，后续语句不再执行。
- 页面可以查看每条SQL的执行结果和耗时。
- 上传同名文件不会覆盖历史执行记录。
- 超过大小限制或编码不正确的文件不能进入预检阶段。

## FR-03 查询结果导出

**功能目标：**
支持用户将SQL查询结果导出为本地文件。

**触发条件：**
用户执行查询并获得结果后，点击结果区的“导出”按钮。

**处理规则：**

- 仅查询成功且存在结果集时显示“导出”按钮。
- 默认导出格式为Excel（`.xlsx`）。
- 支持导出全部结果或勾选指定行导出。
- 用户勾选数据行时，仅导出选中的数据。
- 用户未勾选数据行时，默认导出当前查询返回的全部结果。
- 单次最多导出5,000行，与页面结果展示上限保持一致。
- 导出文件应包含查询结果的字段名称和数据内容。
- 导出文件默认启用密码加密。
- 导出前由用户设置8-20位文件密码，密码不得写入日志。（密码使用管理中心全局密码）
- 文件名默认为“资源名称_数据库名称_导出时间.xlsx”。
- 空值、日期、数字和文本应按照对应的数据类型写入Excel。
- 以`=、+、-、@`开头的文本内容必须按照普通文本处理，避免Excel公式注入。
- 导出过程不得重新执行原SQL，避免数据变化或重复执行高风险查询。

**异常与提示：**

- 当前结果为空：导出按钮不可用。
- 选中行已失效：提示用户刷新查询结果后重试。
- 导出数量超过限制：提示当前允许导出的最大行数。
- 文件生成失败：提示“导出失败，请稍后重试”。
- 文件加密失败：不得生成未加密文件。
- 查询结果包含不支持的数据类型：转换为文本后导出，并在页面提示。

**日志要求：**

保持现在逻辑

**验收标准：**

- 用户可以导出全部结果或选中的结果行。
- 导出文件能够正常打开，字段和数据与页面结果一致。
- 未输入符合要求的密码时不能导出。
- 导出日志可以通过审计编号查询。
- 没有查询结果时不能执行导出。

## FR-04 高危SQL检测与阻断

**功能目标：**
在SQL到达数据库之前识别高风险操作并阻止执行。

**触发条件：**
用户执行单条SQL、多条SQL、“执行全部”或确认执行SQL脚本。

**处理规则：**

- 系统通过词法分析忽略注释内容，不直接修改原始SQL。
- 使用MySQL AST解析器完成语句分割、语句类型识别和条件分析。
- 检测顺序为：资源权限校验→SQL长度校验→语法解析→风险规则检测→数据库执行。
- 单次请求包含多条SQL时，必须对全部SQL完成预检后再执行。
- 任意SQL命中启用的阻断规则时，整个执行请求停止，不向数据库发送任何SQL。
- 阻断弹窗显示风险类型、名单库名称、命中语句序号、所在行号。
- 同一语句命中多条规则时，主页面展示最高风险等级规则，详情中展示全部命中规则。
- 允许用户临时绕过阻断规则，需要通过人脸识别验证后方可放行。
- 用户只能修改SQL后重新执行，或联系管理员调整规则。
- 规则修改只对修改完成后的新执行请求生效。
- 已进入检测流程的请求继续使用开始检测时的规则版本。

**默认阻断范围至少包括：**

- 删除数据库
- 删除数据表
- 清空数据表
- 无有效WHERE条件的DELETE
- 无有效WHERE条件的UPDATE
- 删除字段或索引等高风险表结构修改
- 用户创建、删除及权限变更
- 写入系统数据库
- 写入服务器文件
- 数据库服务控制类命令

**异常与提示：**

- AST解析器无法确定SQL风险时，采用失败关闭策略，不执行SQL。
- 风险检测服务不可用时，不允许执行SQL。
- SQL语法错误时，提示错误位置，不进入数据库执行阶段。
- 同一请求命中超过20条规则时，页面展示前20条并显示命中总数。

**日志要求：**

- 保持现在逻辑

**验收标准：**

- 预置高危SQL测试集全部被阻断。
- 被阻断的SQL在数据库侧不存在对应执行记录。
- 多条SQL中任意一条命中规则时，整个请求不执行。
- 阻断后Web终端会话保持连接。
- 用户修改SQL后可以再次执行。
- 关闭某条规则后，仅该规则不再生效，其他规则保持正常。
- 解析器异常时SQL不会被放行。

## FR-05 SQL黑/白名单与风险规则管理

**功能目标：**
允许管理员查看、导入、启用或停用SQL黑名单及风险规则。

**触发条件：**
管理员进入“SQL黑名单”页面。

**处理规则：**

- 复用现有SQL黑/白名单配置和匹配逻辑。
- Web终端执行SQL时使用与现有功能一致的黑/白名单配置。
- 页面展示：参考现在逻辑
- 支持通过Excel文件批量导入配置。
- 系统提供固定Excel导入模板。
- 导入前校验必填字段、字段格式、重复规则和非法内容。
- 导入完成前必须展示新增、更新、跳过和失败数量。
- 用户确认后才正式写入规则配置。
- 相同规则默认更新现有规则，不创建重复规则。
- 任意一条数据校验失败时，不写入整批数据，并提供错误明细下载。
- 每次规则修改或导入成功后生成新的规则版本号。
- 新规则版本仅影响之后发起的SQL执行请求。
- 普通操作员和审计员仅可查看规则，不可修改。

**异常与提示：**

- Excel格式错误：拒绝导入并提示使用标准模板。
- 必填字段缺失：显示具体行号和字段名称。
- 存在重复规则：提示将更新的规则数量。
- 导入过程中发生异常：整批回滚，不产生部分配置。
- 规则保存失败：页面恢复修改前状态并提示重试。

**日志要求：**

- 保持现在逻辑

**验收标准：**

- 管理员可以查看、启用和停用规则。
- 管理员可以通过标准Excel模板批量导入规则。
- 错误数据不会产生部分导入。
- 普通操作员和审计员无法修改规则。
- 规则修改后，新执行请求使用新版本。
- 历史审计日志仍能查看执行时使用的规则版本。

**触发条件：**
用户建立连接、执行SQL、上传脚本、触发阻断、停止查询、导出结果或结束会话。

**处理规则：**

- 沿用现有审计日志页面、权限和存储逻辑。

  

**日志字段至少包括：**

- 无

**异常与提示：**

- 无

**验收标准：**

- 无

## FR-07 其他配置

### FR-07-01 资产添加

- 沿用现有资产添加、编辑、删除、启用和禁用逻辑。
- 沿用现有数据库账号和密码保存方式。
- Web终端仅允许连接已启用且已授权的数据库资产。
- 数据库密码只能在服务端使用，不得返回前端。
- 资产被禁用后不允许创建新的Web终端会话。
- 已建立的会话按照现有资产禁用策略关闭。

### FR-07-02 应用发布

- 在现有应用发布功能中增加“应用类型”字段。
- 应用类型至少包含“Web终端”和现有应用类型。
- Web终端类型默认发布，无需单独配置。
- 发布后，已授权用户可以通过资源登录入口选择“Web终端”。
