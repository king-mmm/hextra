你好，我是这个博客的作者。

我把这里当作一个长期积累的地方，用来记录技术学习、项目实践、问题排查和一些阶段性的思考。比起写一篇看起来很完整的文章，我更在意把真实遇到的问题、解决过程和复盘沉淀下来。

## 我正在关注

目前我主要关注这些方向：

- 后端开发与工程实践
- Java、Spring Boot 相关技术栈
- 数据库、权限、审计和运维工具
- Web 应用设计与产品化思考
- 自动化、效率工具和个人知识管理

这些内容会慢慢体现在博客文章和项目记录里。

## 这个博客会写什么

这个博客不会只放“最终答案”，我也会记录中间过程。

常见内容包括：

- 学习笔记：把一个知识点讲清楚，方便以后回看
- 项目复盘：记录需求、设计、实现、踩坑和取舍
- 问题排查：从现象到根因，再到解决方案
- 工具使用：整理能提升效率的工具和工作流
- 零散思考：技术之外，但仍然值得保存的想法

如果一篇文章现在还不够成熟，也没有关系。长期来看，持续更新比一次写完美更重要。

## 我的写作原则

- 尽量写清楚背景，而不是只贴结论
- 尽量保留排查过程，而不是只说“怎么做”
- 尽量用自己的语言复述，而不是搬运资料
- 允许文章后续更新，因为理解本来就是逐步变深的

## 项目与作品

我会把值得公开的项目放在[项目](/projects/)页面。

项目页适合放：

- GitHub 仓库
- 在线演示地址
- 技术方案说明
- 项目复盘文章

如果项目暂时不方便公开，也可以先写一篇复盘文章，把关键经验留下来。
